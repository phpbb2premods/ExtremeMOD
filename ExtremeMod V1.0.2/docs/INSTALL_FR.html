1. Installation rapide

Si vous possédez des connaissances basiques de l'utilisation de FTP et que vous avez la certitude que votre serveur ou votre hôte est compatible avec phpBB 2, vous pouvez utiliser cette procédure d'installation rapide. Pour une explication plus détaillée, sautez ce chapitre et consultez la section 2 plus loin.

  1. Décompressez le fichier archive de phpBB 2 dans un dossier de votre système local
  2. Chargez tous les fichiers issus de cette archive (en conservant la structure de répertoires) dans un dossier accessible par le web sur votre serveur ou votre compte chez votre hébergeur.
  3. Modifiez les droits du fichier config.php pour qu'il soit modifiable par tous (666 ou -rw-rw-rw- via votre Client FTP)
  4. Avec votre explorateur web, pointez à l'endroit où vous avez placé phpBB 2 en ajoutant install/install.php, par exemple http://www.mydomain.com/phpBB2/install/install.php, http://www.mydomain.com/forum/install/install.php etc.
  5. Renseignez toutes les informtions demandées et cliquez sur Submit
  6. Modifiez les droits du fichier config.php pour qu'il ne soit modifiable que par vous-même (644 ou -rw-r--r-- via votre Client FTP)
  7. Cliquez sur le bouton Complete Installation ou pointez de nouveau à l'endroit que vous aviez choisi précédemment.
  8. phpBB 2 devrait maintenant être disponible, lisez tout de même la section 8 plus loin pour d'importantes instructions post-installation.

Si vous rencontrez des problèmes ou doutez de la marche à suivre pour une des étapes ci-dessus, consultez le reste de la document.

2. Pré-requis

L'installation de phpBB 2.0.9 nécessite :

  • Un serveur web ou un compte sur un hôte web utilisant un des systèmes d'exploitation majeurs
  • Un gestionnaire de base de données SQL, parmi :
    • MySQL (3.22 ou supérieure)
    • PostgreSQL 7.0.3 ou supérieure (de préférence 7.1.x ou 7.2.x)
    • MS SQL Server (7 ou 2000) en direct ou via ODBC
    • MS Access (2000 ou XP) via ODBC
  • PHP (3.0.9 ou supérieur ou de préférence 4.x exécuté comme un module) avec support de la base de données que vous utilisez

Si votre serveur ou votre compte ne remplit pas les pré-requis ci-dessus, il est à craindre que phpBB 2.0 ne soit pas pour vous.

3. Nouvelles Installations

L'installation de phpBB 2 dépend de votre serveur et votre gestionnaire de base de données. Si vous disposez d'un accès shell à votre compte (via telnet ou ssh par exemple) vous avez la possibilité de télécharger le fichier archive de phpBB 2 (en mode binaire!) dans un répertoire de votre hôte afin de le décompresser.

Si vous n'avez pas d'accès shell ou ne désirez pas l'utiliser, vous devez décompresser le fichier archive de phpBB 2 dans un répertoire local de votre système à l'aide de votre utilitaire favori (winzip, rar, zip, etc.) Ensuite, vous téléchargez via FTP tous les fichiers qui en sont extraits (en gardant la structure de répertoires et les noms des fichiers!) dans le répertoire approprié de votre hôte. Assurez-vous que la casse des noms de fichiers soit inchangée, NE forcez PAS tous les noms de fichiers en majuscules ou minuscules car cela entraînera des erreurs par la suite.

Tous les fichiers se terminant par .php, .inc, .sql, .cfg et .htm doivent être transférés en mode ascii, tandis que les fichiers images doivent l'être en mode binary (binaire). Si vous ne maîtrisez pas ces notions, merci de consulter la documentation de votre utilitaire client ftp. Dans la plupart des cas, tout ceci vous sera rendu transparent par votre utilitaire client ftp mais si vous rencontrez des problèmes par la suite, assurez-vous que les fichiers ont été transférés conformément à ce qui est décrit dans ce chapitre.

La langue standard de phpBB2 est l'anglais. De nombreux packs spécifiques pour des langues différentes sont disponibles. Si vous n'êtes pas anglophone, vous voudrez sûrement installer un ou plusieurs de ces packs avant de continuer. Le processus d'installation ci-dessous vous permet de sélectionner une langue par défaut parmi celles qui sont disponibles (vous pourrez, bien entendu, changer cette langue par défaut plus tard). Pour plus de détails sur les packs de langue, où les trouver et comment les installer consultez le fichier README_FR.

Une fois tous les fichiers téléchargés sur votre site, vous devez faire pointer votre explorateur à l'endroit choisi en ajoutant install/install.php (notez que si vous avez changé l'extension php, consultez la Section 3.i ci-dessous, vous devez modifier install.php de façon appropriée). Par exemple, si le nom de votre domaine est www.mydomain.tld et que vous avez placé phpBB 2 dans le répertoire /phpBB2 situé à la racine, vous devez entrer http://www.mydomain.tld/phpBB2/install/install.php dans l'explorateur. L'écran d'installation de phpBB 2 devrait alors apparaître.

3.i. Changement de l'extension des fichiers

Si l'extension des fichiers utilisés pour exécuter les scripts PHP de votre serveur n'est pas .php mais, par exemple, .php3 ou .phtml vous devez d'abord renommer tous les fichiers se terminant par .php et changer la valeur de $phpEx dans extension.inc. Dans la large majorité des cas, cette étape ne sera pas nécessaire.

3.ii. Installation à partir d'une version phpBB 1.x

ATTENTION, NE placez PAS les fichiers phpBB 2 dans le même répertoire que les fichiers de phpBB 1.x précédemment installée! Créer un nouveau dossier (ou déplacez l'ancien phpBB 1.x), ne pas le faire vous expose à des erreurs d'exécution (runtime errors).

3.iii. Installation avec MySQL 3 et 4, PostgreSQL 7.x, MS SQL 7/2000 (y compris via ODBC)

Avant de commencer l'installation, assurez-vous de disposer d'une base de données fonctionnelle. La plupart des hébergeurs mettent maintenant à disposition au moins une base de données dans leur offre de base mais elle nécessite souvent une configuration avant de pouvoir être utilisée. Adressez-vous à votre hébergeur pour de plus amples renseignements. Si vous disposez de votre propre serveur et que vous n'êtes pas sûr(e) de la façon de créer votre propre base de données, consultez la documentation de votre gestionnaire de base de données ou le site web correspondant, par exemple http://www.mysql.com/doc/, http://www.postgresql.org/users-lounge/, etc.

ATTENTION, si votre base de données est MySQL, veuillez sélectionner la version appropriée dans la boîte de sélection. NE sélectionnez PAS MySQL 4.x si vous avez MySQL 3.x (la version la plus répandue). En cas de doute, sélectionnez MySQL 3.x.

3.iv. Installation avec MS Access 2000/XP

Avant d'installer phpBB 2 vous devez créer un Nom de Source de Données ou DSN(Data Source Name). La procédure exacte pour le faire dépend de votre hébergeur, prenez contact avec lui en cas de doute avant de commencer. En général, vous devriez créer un DSN système qui pointe où vous avez stocké une copie décompressée du fichier ms_access_primer.mdb ( ms_access_primer.zip peut être trouvé dans db/schemas/). Le nom que vous donnez à ce DSN sera utilisé durant l'installation. Ne poursuivez pas l'installation si cette étape n'est pas réalisée !

Veuillez noter que MS Access n'est pas conseillé pour une utilisation intensive. Si vous envisagez que plus d'un ou deux utilisateurs puissent être connectés simultanément à chaque instant, vous devriez vous orienter vers une solution plus robuste.

3.v. Saisie des données nécessaires

Une fois que vous avez atteint la page d'installation, vous avez à saisir plusieurs champs. Assurez-vous de sélectionner le type de base de données correct ! Si vous ne disposez pas du nom de base de données, serveur de base de données, etc. contactez votre hébergeur. Pour les bases de données non-ODBC le serveur de base de données est en général localhost mais ce n'est pas standard! Si vous utilisez une base de données avec ODBC (comme MS Access ou MSSQL via ODBC) il faut saisir le DSN (cf. 2.iv.).

Le préfixe de base de données vous permet d'entrer quelques caractères, un nom court, etc. qui préfixera le nom de toutes les tables utilisées lors de l'installation. La valeur par défaut (phpbb_) est en général suffisant. Si vous disposez de plusieurs copies de phpBB 2 utilisant la même base de données, modifiez ce paramètre ou vous risquez des erreurs lors de l'installation.

Les informations suivantes sont propres à votre site. Vous devez entrer le nom d'utilisateur et le mot de passe du compte administrateur initial (des administrateurs supplémentaires pourront être créés plus tard). Le nom du serveur, le port du serveur et le chemin de script sont essentiels parce que tous les emails contenant des URLs utiliseront ces paramètres. Le programme d'installation essaye de deviner les valeurs initiales mais elles peuvent être incorrectes, alors vérifiez-les !

Souvenez-vous qu'un nom de domaine (ou de serveur) ne contient pas http:// ! Ne saisissez que le nom de domaine, par exemple www.mydomain.tld et pas http://www.mydomain.tld/. Le port du serveur peut, dans la grande majorité des cas, être laissé à 80, qui est le port http standard. Mais si votre serveur web est connecté à un port différent parce que, par exemple, le port 80 est verrouillé, modifiez ce paramètre en conséquence. Le chemin du script est le chemin relatif depuis la racine où phpBB 2 est installé, par exemple si phpBB 2 est installé dans phpBB2/ sous la racine, alors entrez phpBB2 (ou /phpBB2/) comme chemin de script.

3.vi. Fin de l'installation

Une fois que vous avez vérifié toutes les données à saisir, cliquez sur "submit". Le script d'installation crée et alimente toutes les tables nécessaires. L'installation essaye alors d'écrire dans le fichier config.php qui contiendra alors la configuration de base pour exécuter phpBB 2. Si le script d'installation ne peut écrire dans config.php directement, il vous est possible de télécharger le fichier ou d'utiliser FTP pour le mettre à l'emplacement adéquat. Si vous choisissez de télécharger le fichier, il vous faut le recharger à l'emplacement du config.php existant sur votre serveur. Si vous essayez d'utiliser FTP via le script d'installation vous aurez à fournir des informations indispensables. Si FTP échoue vous pourrez télécharger le fichier et le recharger comme décrit précédemment.

Pendant la procédure d'installation de phpBB 2 une vérification de la disponibilité du module de base de données de PHP est effectuée et le processus s'arrête si ce n'est pas le cas. Pour éviter cela, vérifiez que vous avez sélectionné la base de données correcte et/ou demandez conseil à votre hébergeur.

Ne poursuivez pas avant de terminer correctement l'installation et (si nécessaire) d'avoir recharger le fichier config.php mis à jour.

4. Mise à jour depuis phpBB 1.4.x

Avant toute mise à jour, nous vous recommandons fortement de faire une sauvegarde complète de votre base de données ! Si vous n'êtes pas sûr(e) de la procédure à suivre pour le réaliser, demandez conseil à votre hébergeur. Une fois votre base de données sauvegardée, vous pourrez suivre les instructions données dans le chapitre Nouvelles Installations plus haut. Mais vous devez sélectionner Mise à jour au lieu de Installation comme Méthode d'installation.

Comme l'installation, la mise à jour est automatisée. Le fichier config.php est mis à jour automatiquement (si possible, sinon il vous est demandé de télécharger une copie comme dans le chapitre Nouvelles Installations). Une fois le fichier config.php en place, tout devrait se dérouler sans erreur. Veuillez noter que le processus de mise à jour peut prendre du temps et dépend de votre hébergement (un échec peut être dû au serveur web utilisé ou à des timeouts). Si tel est le cas contactez votre hébergeur pour lui demander d'allouer exceptionnellement plus de ressources au script d'installation (soyez poli(e) et ils vous seront d'une aide précieuse).

Une fois l'opération terminée, votre forum sera disponible immédiatement. Si vous rencontrez des erreurs, rechargez votre base de données d'origine et faites part du problème à notre "bug tracker" (voir le fichier README_FR pour plus de détails).

5. Mise à jour depuis phpBB 2.0 beta-1

Désolé, mais aucune mise à jour directe n'est disponible pour la beta-1 (on vous avait prévenu!). Si vous êtes à l'aise avec SQL, etc. vous devriez pouvoir effectuer les changements dans la base de données manuellement. Vous aurez besoin d'examiner les CVS check-ins pour déterminer où et quoi modifier.

6. Mise à jour depuis une version de phpBB 2 Release Candidate

La mise à jour depuis une version RC est en général plus facile. Tout d'abord, faites une sauvegarde de votre fichier config.php existant, et gardez-la en sécurité ! Ensuite, effacez tous les fichiers phpBB 2 existants, sans en oublier un seul, sinon vous risquez de rencontrer des erreurs ultérieurement. Vous pouvez laisser les templates ajoutés mais notez qu'ils ne fonctionneront peut-être pas dans la version finale. Il est de plus recommandé de revenir à subSilver si vous utilisez un style différent. Ensuite, vous pouvez charger les nouveaux fichiers de phpBB 2 (voir le chapitre Nouvelles Installations pour plus de détails). Une fois la copie terminée, écrasez le fichier config.php généré par l'installation avec votre sauvegarde.

Vous pouvez maintenant exécuter le fichier install/update_to_208.php qui, selon la version précédente que vous utilisiez, procédera aux modifications de la base de données. Vous obtiendrez peut être des FAILURES pendant la procédure, qui peuvent être ignorés sauf si vous voyez ERROR, qui provoquera l'arrêt du script (dans ce cas, vous pouvez obtenir de l'aide via nos forums ou notre "bug tracker").

Utilisateurs de la version RC-2 (et précédentes) et de MSSQL, notez que pendant la procédure de mise à jour, les tables de votre forum seront supprimées et re-créées. Toutes les données des champs standards seront préservées. Si vous avez modifié la table forums, ajouté des champs ou modifié des champs existants, ces changements seront PERDUS. Si cette issue est pénalisante pour vous, nous vous recommandons de mettre en commentaire la section "DROP FORUM TABLE" du fichier update_to_208.php et de manuellement la remplacer par des instructions qui modifient la colonne forum_id pour supprimer la valeur d'IDENTITY (si besoin).

Une fois le script update_to_208 terminé, vous DEVEZ accéder au panneau de configuration générale de l'Administration (Administration General Configuration panel) et vérifier les données qui s'y trouvent. C'est essentiel si vous utilisiez une version antérieure à la RC-3 puisque des informations supplémentaires doivent être entrées pour permettre l'envoi correct d'email contenant des URLs.

6.i. Changements du format de pack de langues et des templates

Notez que des modifications ont été apportées dans le format des langues, de la localisation des images et dans les templates depuis les précédentes RC. Si vous utilisiez une version antérieure à la RC-3, vous devez charger les nouveaux packs de langues et de localisation d'images. (language and image localisation packs). Vous pouvez sans crainte supprimer tous les fichiers *_lang.gif existants dans le répertoire subSilver/images puisqu'ils ne sont plus utilisés. Si vous utilisiez ou aviez installé un template d'une tierce-partie, vous devez soit l'enlever soit le mettre à jour. Quelques variables de templates ont été modifiées, ce qui peut entraîner des dysfonctionnements dans ces templates tierces sauf s'ils suivent les standards de phpBB 2.0.x.

7. Mise à jour depuis une précédente version stable de phpBB 2

Si vous utilisez actuellement une précédente version stable de phpBB 2, la mise à jour est directe. Vous devez télécharger un des trois packs et votre choix détermine ce que vous devez faire. Attention: avant de mettre à jour, nous recommandons vivement de faire une sauvegarde complète de votre base de données et des fichiers phpBB2! Si vous n'êtes pas sûr de la manière de procéder pour le faire, demandez conseil à votre hébergeur.

Utilisateurs de MySQL, veuillez noter: Des modifications introduites dans la version 2.0.4 peuvent accroître la taille de votre base de données, plus spécialement la table search_wordmatch. D'autres changements dans cette version ont pour but de réduire la taille de la base de données. Mais ceux-ci ne sont pas applicables à tous les types de forums (particulièrement les forums non-anglais). Bien que l'accroissement de taille sera relativement réduit, cela peut être pénalisant pour certains utilisateurs. En particulier ceux dont l'hébergement limite la taille de base de données et qui possèdent déjà une taille de base à la limite du maximum.

7.i. Pack complet (Full package)

Si vous avez téléchargé le pack complet, suivez la même procédure générale de mise à jour que pour "Mise à jour depuis une version de phpBB 2 Release Candidate". Sauf en ce qui concerne le besoin de changer ou mettre à jour les styles. De plus, il faut en général vérifier les données du panneau de contrôle d'administration (c'est en tout cas conseillé)

7.ii. Fichiers modifiés seulement (Changed files only)

Ce pack contient un certain nombre d'archives, qui contiennent chacun les fichiers modifiés à chaque version jusqu'à la 2.0.9. Vous devez sélectionner l'archive correspondant à votre version actuelle, par exemple si vous êtes actuellement en 2.0.5, sélectionnez le fichier phpBB-2.0.5_to_2.0.9.zip/tar.gz

La structure des répertoires a été préservée afin de vous permettre (si vous le désirez) de simplement charger le contenu de l'archive au même endroit sur le serveur, c'est à dire d'écraser les fichiers existants avec les nouvelles versions. N'oubliez pas que si vous avez ajouté des mods ces fichiers pourront écraser des fichiers modifiés. Vous devrez réappliquer les mods à chacun des fichiers concernés.

Comme pour les autres procédures de mise à jour, vous devez exécuter install/update_to_208.php après avoir mis à jour les fichiers. Cela mettra à jour le schéma et les données (si nécessaire) de la base de données et incrémentera le numéro de version en base.

7.iii. Fichier Patch (Patch file)

Le fichier Patch est probablement la meilleure solution lorsqu'on dispose de nombreux modules ou modifications et qu'on ne veut pas les réappliquer à tous les fichiers modifiés. Pour utiliser cette méthode, vous avez besoin d'un accès à la ligne de commande pour utiliser l'application standard UNIX patch.

Des fichiers patches sont fournis pour permettre la mise à jour depuis chaque version stable précédente. Sélectionnez le patch adéquat, par exemple si votre version actuelle est 2.0.2 vous devez choisir phpBB-2.0.2_to_2.0.9.patch. Placez le patch sélectionné dans le dossier contenant les fichiers de phpBB 2 (c'est à dire index.php, viewforum.php, etc.). Ensuite, exécutez la commande suivante: patch -p0 < [PATCH NAME] (où PATCH NAME est le nom du fichier de patch choisi). Le processus se terminera rapidement, et on l'espère sans aucun HUNK FAILED.

Si vous obtenez des échecs, jetez un oeil sur la méthode Fichiers modifiés seulement (Changed files only) pour remplacer les fichiers en question, avec la condition de réappliquer manuellement tous les mods sur ces fichiers. Mais, vous pouvez également examiner les fichiers .rej pour déterminer les raisons des échecs et effectuer quelques ajustements aux sources incriminés.

Vous devez bien entendu supprimer le fichier Patch une fois que vous l'avez utilisé. Comme pour les autres procédures vous devez exécuter install/update_to_208.php après avoir fini la mise à jour des fichiers. Cela mettra à jour le schéma et les données (si nécessaire) de la base de données et incrémentera le numéro de version en base.

7.iv. Tous types de packs (All package types)

Si vous disposez de packs de langues différents de l'anglais, vous devez vérifier si une nouvelle version est disponible. Un certain nombre de phrases manquantes ont été ajoutées, ce qui, même si ce n'est pas essentiel, profitera à vos utilisateurs. Notez que tous les packs de langues n'ont pas été mis à jour, préparez vous à vérifier périodiquement leur disponibilité.

8. Tâches importantes à réaliser après toute installation (relatives à la sécurité)

Une fois phpBB 2.0.9 installé, vous DEVEZ supprimer les dossiers /install et /contrib . Les laisser en place est une très sérieuse faille de sécurité potentielle qui peut aboutir à l'effacement ou l'altération de vos fichiers, etc. Notez que jusqu'à ce que ces dossiers soient effecés, phpBB2 ne fonctionnera pas et un message d'avertissement sera affiché. En plus de ces suppressions essentielles vous pouvez également effacer le dossier docs/ si vous le désirez.

Une fois ces dossiers effacés, accédez au panneau d'administration (administration panel) L'accès à ce panneau dépend de la fin de l'installation qui peut vous y diriger automatiquement. Si ce n'est pas le cas, connectez vous en tant qu'administrateur (celui que vous avez spécifié pendant l'installation/mise à jour) et cliquez sur le lien "Administration Panel" en bas de n'importe quelle page. Assurez-vous que les informations affichées dans General -> Configuration sont correctes !

8.i. Avatars uploadables

phpBB 2 possède plusieurs méthodes pour permettre aux utilisateurs de sélectionner leur propre avatar (un avatar est une petite image, généralement associée à un utilisateur, et affichée juste sous son nom dans les messages).

Deux de ces options permettent aux utilisateurs de charger leur avatar depuis leur propre machine ou une localisation distante (via une URL). Si vous désirez activer ces possibilités vous devez d'abord vous assurer que les chemins de chargement des avatars (paths for uploadeable avatars) spécifiés dans Admin -> General -> Configuration -> Avatars sont corrects. Par défaut, il s'agit de images/avatars mais vous pouvez librement le modifier, à condition de mettre à jour la configuration. Vous devez également vous assurer que l'écriture dans ce répertoire est possible pour le serveur web. Habituellement, cela nécessite de modifier les permissions pour autoriser les lectures et écritures pour tout le monde dans ce répertoire. La procédure exacte pour le faire dépend de votre client ftp et de votre système d'exploitation.

Sous UNIX par exemple, positionnez le répertoire à a+rwx (ou ugo+rwx ou même 777). Utilisez la ligne de commande et chmod ou bien votre client ftp (commande "Change Permissions", "chmod" ou équivalent, consultez la documentation de votre client ftp pour les détails). La plupart des clients FTP listent les permissions pour le user (Read, Write, Execute), Group (Read, Write, Execute) et Other (Read, Write, Execute). Cochez toutes les cases pour permettre un accès correct.

Sous Windows, vous devez vous assurer que le répertoire n'est pas protégé contre l'écriture et qu'il possède des permissions d'écriture globales (consultez la documentation serveurs ou contactez votre hébergeur si vous n'êtes pas habitué(e) à ces commandes).

Attention, soyez averti(e) que positionner un répertoire avec un accès en écriture global est une faille de sécurité potentielle. Même si rien de critique ne peut se produire (sauf l'effacement de tous les avatars) il y a toujours des personnes qui peuvent provoquer ce genre d'ennuis. Vous devez donc surveiller ce répertoire et en effectuer des sauvegardes régulières.

8.ii. Safe Mode

phpBB 2.0.9 inclut un support pour l'utilisation des avatars uploadables sur les systèmes exécutant PHP en mode protégé (safe mode). Si cela correspond à votre système, vous devez créer un sous-répertoire tmp dans le répertoire spécifié pour le stockage des avatars chargés (par défaut il s'agit de images/avatars). Donnez-lui les mêmes permissions d'accès que spécifié ci-dessus.

Le support du safe mode est compatible avec diverses restrictions qui peuvent vous être imposées (si elles ne sont pas trop restrictives et si la version de PHP installée est une 4.0.3 ou supérieure). Il n'y a généralement pas besoin de configuration manuelle pour le support du safe mode qui est normalement transparent.

8.iii. Options supplémentaires

Ce package inclut deux fichiers supplémentaires (optionnels), que vous pourrez trouver dans le répertoire contrib/. Ces options sont les fichiers template_file_cache.php et template_db_cache.php qui concernent les temps de chargement et de génération des pages sur les systèmes lents ou surchargés. Ces modules mis à jour compilent chaque template et utilisent les versions compilées de ces fichiers plutôt que de les générer à chaque demande de visualisation. Les tests ont semblé démontrer une baisse significative des temps de génération des pages et plus important, une baisse significative de la charge du serveur en environnements d'hébergement mutualisé (virtual hosting environments).

Consultez le fichier README_FR contenu dans le dossier contrib pour plus de détails.
