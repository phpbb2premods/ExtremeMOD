1. Installation de phpBB 2

Les instructions détaillées d'installation et de mise à jour peuvent être consultées dans le fichier INSTALL_FR contenu dans cette distribution. Si vous désirez mettre à jour une version antérieure à phpBB 1.4.x nous vous recommandons vivement d'effectuer une sauvegarde de vos données actuelles avant de démarrer !

Attention, les utilisateurs de versions Release Candidate (RC) de phpBB 2 doivent exécuter le script update_to_205.php script AVANT de tenter d'accéder à leur forum. Ne pas le faire vous expose à des erreurs et comportements inattendus ! Bien que cela ne provoque pas de dommages, vous ne pourrez pas utiliser votre forum correctement.

Si vous utilisez la version RC-1 (pre) vous pourrez voir que quelques changements ont été faits au niveau Base de données et vous devez examiner le schéma pour décider si votre installation nécessite des modifications (notez que tout ceci a déjà été évoqué sur les forums phpBB 2 et est pris en compte dans le script de mise à jour). Si vous ne vous sentez pas assez confiant(e) pour faire ces manipulations, nous vous recommandons une ré-installation (sauvegardez les données actuelles si vous le désirez, re-installez phpBB 2 et rechargez votre sauvegarde).

Les utilisateurs de phpBB 2.0 Beta-1 ne peuvent mettre à jour directement leur version en phpBB 2.0 RC-2, 3 ou 4, nous en sommes désolés.

2. Exécution de phpBB 2

Une fois phpBB 2.0 installé, il est facilement administrable par des panneaux de contrôles pour les administrateurs (admin) et les modérateurs (moderator). Si vous avez besoin d'aide ou de conseils au sujet de phpBB 2, consultez la Section 3.

2.i. Internationalisation, i18n

De nombreux packs de langues (language packs) et de versions locales de SubSilver (subSilver localisations) sont maintenant disponibles. Vous pouvez les trouver sur notre page officielle de téléchargement:

http://www.phpbb.biz/download (site francophone)
http://www.phpbb.com/downloads.php (site anglophone)

Il s'agit de la distribution Officielle de tous les supports de langues. Si vous chargez un package d'une tierce-partie, comprenez que nous n'en assurons aucun support. Dans ce cas, merci de ne pas nous contacter à ce sujet !

L'installation de ces packages est directe, il suffit simplement de charger le pack de langue désiré et de le désarchiver dans le dossier languages/ . Assurez-vous de conserver la structure des répertoires lorsque vous procédez à cette opération ! Pour installer les packs d'images subSilver, désarchiver les fichiers dans le dossier templates/subSilver/images , en conservant la structure des répertoires. Une fois installées les langues sont disponibles immédiatement.

Si votre langue n'est pas encore disponible, consultez nos forums où vous trouverez un topic listant les traductions disponibles ou en cours de préparation. Ce topic décrit également les informations pour se porter volontaire à la traduction dans une langue non encore listée.

Attention les utilisateurs qui mettent à jour en 2.0.9 des versions antérieures à la RC-3 doivent télécharger de nouvelles versions des packs d'images de language/subSilver . Les packs chargés de versions précédentes à la RC-3 ne fonctionneront pas correctement avec cette version de phpBB 2.

Si vous désirez mettre à jour phpBB 2.0.0 et que vous utilisez un pack de langue différent de l'anglais, vous pourrez bénéficier de versions mises à jour à télécharger très prochainement. Elles disposeront de quelques chaînes de caractères absentes de la première version et de quelques ajouts et modifications.

2.ii. Styles

Bien que le phpBB Group soit fier du style subSilver (qui a influencé nombre de designs d'autres forums concurrents) nous sommes conscients qu'il n'est pas au goût de tout le monde. Ainsi, phpBB 2 permet le changement de style très facilement. Tout d'abord, téléchargez un style qui vous plait. Nous en mettons à disposition sur

http://styles.phpbb.biz/ (site francophone)
http://www.phpbb.com/styles/ (site anglophone)

Attention les styles obtenus auprès de tiers pour des versions antérieures à phpBB2 2.0.x peuvent ne pas fonctionner correctement. Il est conseillé de les retirer ou de les mettre à jour (si possible) dans une version compatible avec phpBB 2.0.x .

Une fois que vous avez chargé un style, l'étape suivante consiste à le désarchiver (ou charger le contenu non compressé) dans le dossier templates/ . Ensuite, accédez à la fonction Administration -> Styles -> Add, vous devriez voir que votre nouveau style est présent, cliquer sur add et il sera alors disponible pour tous vos utilisateurs.

2.iii. Modules (Mods)

Bien que non officiellement supportés par le phpBB Group, phpBB 2 possède un système de modules (connus sous les noms de hacks ou mods). Ces modifications effectuées par des tiers sur le standard phpBB 2 permettent d'en étendre les fonctionnalités et peuvent être trouvés à :

http://mods.phpbb.biz/ (site francophone)
http://www.phpbb.com/mods (site anglophone)

Attention, tout bug ou erreur se produisant après un ajout de module ne doit pas être signalé au bug tracker (voir plus loin). Retirez d'abord le module et regardez si le problème est résolu.

Souvenez-vous aussi que tout module modifiant la base de données peut rendre plus difficile la mise à jour de votre forum vers des versions futures. Malgré ces limites, beaucoup d'utilisateurs utilisent avec succès la plupart des modules disponibles.

3. Obtenir de l'aide sur phpBB 2

phpBB 2 peut sembler un peu obscur aux nouveaux utilisateurs, particulièrement le système de permissions. La première chose à faire est de consulter notre FAQ_FR qui couvre quelques questions basiques de démarrage. Si vous avez besoin d'aide supplémentaire, il existe plusieurs endroits pour cela.

3.i. Guide d'utilisation de phpBB 2

Un guide d'utilisation est maintenant disponible en ligne et peut être consulté à cet endroit :

http://forum.phpbb.biz/viewtopic.php?t=4 (guide francophone)
http://www.phpbb.com/guide/phpBB_Users_Guide.html (guide anglophone)

Il couvre l'ensemble des fonctionnalités, de l'installation jusqu'au système de permissions et la gestion des utilisateurs.

3.ii. Forums de la communauté

Le phpBB Group dispose d'une communauté active où des personnes ont décidé généreusement de donner leur temps pour aider les utilisateurs. Notre site se trouve à:

http://www.phpbb.biz/ Site de la première communauté francophone phpBB
http://www.phpbb.com/ Site officiel (en anglais)

Si vous désirez de l'aide via nos forums, utilisez la fonction de Recherche (Search) avant de poster votre question. Cela permet de ne pas perdre votre temps, ni le nôtre et aux développeurs et modérateurs de se consacrer aux nouveaux problèmes rencontrés par les utilisateurs. Souvenez-vous également que phpBB est le fruit de volontariat, personne ne reçoit de compensation pour le temps qu'il y consacre, ni les modérateurs, ni les développeurs. Merci de ne pas oublier la patience, le respect et l'attention lorsque que vous attendez une réponse.

3.iii Discussion via IRC (Internet Relay Chat)

Vous pouvez également trouver de l'aide sur notre canal IRC (IRC channel). Celui-ci se trouve sur le réseau IRC Openprojects, irc.openprojects.net le nom du canal est #phpbb et l'accès y est possible par tout bon client IRC comme mIRC, XChat, etc. Encore une fois, merci de ne pas abuser de ce service et de rester respectueux(se).

4. Statut de cette version

Il s'agit de la deuxième distribution de phpBB 2 à être déclarée stable. Les versions 2.0.x possèdent des fonctionnalités figées, les versions à venir corrigeront des bugs ou dysfonctionnements. Notre prochaine version majeure sera phpBB 2.2 dont le développement est en cours (la version non stabilisée de développement est la 2.1). Merci de ne pas nous demander quand la version 2.2 sera disponible, aucune date n'est encore fixée. Vous êtes libres de nous suggérer des fonctionnalités nouvelles pour phpBB 2.2, nous en avons déjà quelques unes en prévision qui peuvent être consultées sur notre gestionnaire de tâches (task tracker):

http://sourceforge.net/pm/task.php?group_project_id=13524&group_id=7885

Nous ne garantissons pas que tout sera disponible en version 2.2 mais nous faisons tout notre possible ! Si votre idée n'apparaît pas dans cette liste, accédez à notre liste de demandes de fonctionnalités (feature request list) :

http://sourceforge.net/tracker/?atid=357885&group_id=7885

Avant de nous soumettre votre idée, assurez-vous d'avoir parcouru la totalité de cette liste. Si votre idée est équivalente à une suggestion existante, ajoutez un commentaire à celle-ci mais n'ajoutez pas une nouvelle suggestion similaire!

Pour certaines idées, nous pourrons demander votre avis via nos forums. Pour ceux que le développement de phpBB 2.2 intéresse, gardez un oeil sur nos forums (particulièrement le forum de discussion des fonctionnalités (Feature Discussion forum)) où vous pouvez avoir une influence sur l'orientation de certaines fonctionnalités. Pour finir, vous pouvez également visiter notre forum de développement afin d'en connaître la progression :

http://area51.phpbb.com/phpBB 2/

Merci de ne pas utiliser ces forums dans le but d'obtenir un support ou poser des questions sur phpBB 2.0.x, les forums principaux de la communauté sont là pour ça. Tout message de ce type sera de toute façon verrouillé et restera sans réponse.

5. Signalement de Bugs

Le phpBB Group utilise un système de gestion de bugs pour stocker, lister et gérer tous les bugs signalés, qui peut être consulté à l'adresse ci-dessous. Merci de NE PAS signaler de bugs sur nos forums, ils seront bloqués.

http://www.phpbb.com/bugs/

Bien que nous apprécions beaucoup de recevoir une description de bug (plus nous aurons de signalements et plus nous augmenterons la stabilité de phpBB 2) nous vous demandons de franchir quelques étapes avant tout signalement:

  • Tout d'abord, soyez sûr(e) que votre bug est reproductible, comment le déterminer dépend du bug en question. Seulement si le bug s'avère reproductible alors il peut s'agir d'un problème dû à phpBB 2.0 (ou en tout cas qui lui est lié). Si ce n'est pas reproductible, alors il peut s'agir de votre hébergeur effectuant une opération, d'un utilisateur ayant une utilisation incohérente, etc. Les signalements de bugs non reproductibles nous empêchent de nous concentrer sur la résolution de réels problèmes reproductibles.
  • Ensuite, merci de consulter et rechercher parmi les rapports existants (souvenez-vous de vérifier les rapports ouverts ET les rapports fermés !) si votre bug (ou un similaire) n'est pas déjà signalé. Dans ce cas, merci d'ajouter vos informations plutôt que de créer une nouvelle entrée en doublon (tout ceci nous ralentit).
  • Consultez les forums (utilisez la recherche !) au cas où d'autres personnes n'auraient pas déjà discuté d'un sujet similaire au vôtre. Dans tous les cas, comme dit précédemment, merci de NE PAS poster votre bug spécifique sur le forum s'il n'est pas reproductible ou si vous n'êtes pas sûr(e) qu'il provienne de phpBB 2.0
  • Si aucun bug existant ne correspond, alors vous êtes libre de référencer le vôtre

Si vous postez un nouveau bug (c'est à dire un bug non déjà référencé), soyez certain(e) d'être connecté(e) (avec le même compte que des forums de la communauté) et merci d'inclure les informations suivantes dans votre rapport :

  • Type et version de votre serveur, c'est à dire Apache 1.2.22, IIS 4, Sambar, etc.
  • Version et mode d'utilisation de PHP, c'est à dire PHP 4.1.1 comme module, PHP 4.0.1 par CGI, etc.
  • Type et version de votre SGBD, c'est à dire MySQL 3.23.32, PostgreSQL 7.1.2, MSSQL Server 2000 SP1, etc.

Merci également d'effectuer un rapport le plus détaillé possible, en décrivant précisément les étapes nécessaires pour reproduire le problème. Si vous avez une solution au problème dont vous êtes ABSOLUMENT SUR(E) (et qui est compatible avec nos guides de développement) sans introduire d'autres problèmes ou incompatibilités, alors portez-la à notre connaissance. Mais ne le faites que si vous jugez que c'est utile, si nous en avons besoin, nous vous le demanderons de toute façon.

Une fois qu'un bug nous est remonté, il vous sera demandé par email des informations complémentaires. S'il vous plait, si nous vous demandons de nous fournir de tels renseignements, faites-le ! Il est extrêmement frustrant pour nous de recevoir des rapports de bug, de demander des informations complémentaires et de n'obtenir aucune réponse. Dans ce cas, notre politique est de fermer le bug, ce qui peut laisser un problème réel sans solution. Évidemment, nous préférerions ne pas en arriver à une telle extrémité.

5.i. Bugs de sécurité

Si vous détectez une faille potentielle dans la sécurité de phpBB 2, merci de NE PAS la référencer dans le bug tracker, un forum ou une mailing list ! Le faire permettrait à des personnes peu scrupuleuses d'en tirer avantage avant que nous ayons pu sortir un correctif. Tout bug relatif à la sécurité doit être signalé directement à un ou plusieurs de nos développeurs.

Cela peut se faire selon une des trois possibilités suivantes : email security@phpbb.com redirigé vers tous les développeurs, message privé (PM) à un des développeurs (consulter la liste des membres du groupe Developer pour connaître le nom des développeurs) ou un email direct (soit en utilisant leur adresse email si elle est disponible, soit à username@phpbb.com où username est le nom du développeur). Si vous utilisez l'email ou le message privé, contactez plusieurs développeurs, un seul ne suffit pas. Sans réponse après un délai raisonnable (un jour ou deux), essayez de contacter d'autres développeurs.

6. Liste des bugs référencés

Cette liste n'est pas complète (voir le lien plus loin pour la liste complète) mais indique les bugs qui sont susceptibles de toucher une large population d'utilisateurs. Les autres bugs listés dans le tracker sont typiquement limités à certaines configuration ou installations/mises à jour.

  • Les quatre premiers caractères du contenu de certains messages disparaissent - il s'agit d'un problème PHP et non phpBB
  • Impossibilité de poster de gros messages avec PostgreSQL 7.0.x (limite de Postgresql 7.0.x)
  • Impossibilité de poster de gros messages avec la configuration par défaut de MSSQL (nécessite une configuration de MSSQL)
  • L'utilisation de charactères non-latin avec MSSQL peut aboutir à des coupures inopinées dans les messages
  • Echec des mises à jours pour certains gros forums sous certains hôtes
  • Problèmes persistants de compatibilité avec PHP 3, nous recommandons une mise à jour en PHP4 le plus tôt possible ( phpBB 2.0.x sera la dernière version supportant PHP 3 )

7. Problèmes de compatibilité avec PHP

Veuillez noter que l'exécution d'une application utilisant une version de développement de PHP peut donner des résultats étranges ou inattendus apparaissant comme des bugs de l'application (ce qui peut être faux). Ainsi, nous vous recommandons de mettre à jour PHP dans la version la plus récente et la plus stable avant d'exécuter phpBB 2. Si vous utilisez une version de PHP de développement, merci de vérifier que les bugs constatés se produisent également sur un système disposant d'une version stable avant de nous en faire part.

Ce forum a été développé et testé sous Linux et Windows (comme les autres) équipés de IIS et Apache connectés à MySQL 3.23 et 4.0.0, MSSQL Server 2000, MS Access 2000 et XP et PostgreSQL 7.x. Les version PHP allant de 4.0.6 à 4.2.3 ont été utilisées sans problème. Il peut y avoir des problèmes non confirmés avec PHP 4.0.1 et les versions patchées (patch level releases). Si l'utilisation de phpBB avec cette version de PHP vous pose problème, nous vous recommandons de mettre cette dernière à jour avant tout signalement de bug.

7.i. Note sur les failles de sécurités de PHP

A la fin du mois de février 2002 une faille de sécurité majeure a été détectée sur PHP 3 et 4, ayant pour conséquence la désactivation de l'upload de fichiers chez certains hébergeurs (bien que des patches correctifs soient disponibles pour la dernière version de PHP 4).

En conséquence, et pour contourner la situation où les hébergeurs refusent l'upload de fichiers, nous avons ajouté plusieurs vérifications dans phpBB 2. Ces vérifications vous empêchent d'activer ou utiliser des fonctions qui ne fonctionneront pas correctement sur le PHP installé sur votre serveur. Actuellement, sont disponibles l'upload des avatars et la restauration de base de données.

Des modules (mods) additionnels peuvent également être affectés mais nous n'en avons aucun contrôle, contactez l'auteur du module pour plus d'informations.
